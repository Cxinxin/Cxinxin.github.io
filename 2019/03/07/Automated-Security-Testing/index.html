Automated Security Testing

🎋 前记

   一不小心又过去了两个月…很多东西来不及整理，就先姑且让它静默在我的脑海里吧🙉，待哪天心血来潮再记录一刻

   话说已经入职新公司一个月了，原本以为会有啥不适之感，事实证明这种事不存在的，无缝对接…我的 Line manager 是山东人，沧桑脸却有着少年般的明眸，性格也十分爽快，明眼一看就知道是个乐天派人士。最近他的 paper 被选中，要开始着手尝试搭建一套 DevSecOps 架构，构想的四大部分其实已经有完成了一些，他交代给我去研究的就是其中一个 OWASP ZAP,一个 Open Source 的 Web 安全工具。公司的网络实在令人烦躁😾,各种限制各种 Block，无奈只能在自家电脑搭建虚拟机来搞…

   💬 这里主要是结合 Selenium 和 OWASP ZAP 做的一个集成测试 Demo

🎋 Prerequisite

  1. Vagrant - Ubuntu box
  2. JDK >= 1.8 ZAP 安装需要 jdk1.8 以上版本
  3. Python / pip
  4. OWAPS ZAP
  5. SeleniumBase

🎋 Why?

   嗯其实 OWASP ZAP 这东东我从来也没了解过，Selenium 就只知道是个自动化测试工具，也无接触经验，说要结合这两个东西我一开始也一头雾水😓。看了不少文章和资料，算是明白了个大概。简单理解，OWASP ZAP 主要是用来做渗透式测试和挖掘漏洞的，通过扫描，自动攻击，爬虫等方式，帮你检测到你的网站是否有什么”隐患”⚡。至于为什么要集成 Selenium，是因为 ZAP 有时候并不能获取到所有的网页资源。想象一下如果我们通过登录认证进入网页，这种需要人为手动操作认证后，才能获取下一步的网页资源，ZAP 可能就没有捕获到。所以需要给 ZAP 一些指引，尽可能 cover 到所有用户流程会操作的网络资源，这时候用自动化测试框架就很方便了，如果有测试人员已经模拟了很多 functional automation test，两者结合使用，ZAP 就能了解整个 web 的 flow。

🌱参考资料：
1.  Using OWASP ZAP, Selenium, and Jenkins to automate your security tests
2.  Driving OWASP ZAP with Selenium

📝 Demo
